2. NLP (Natural Language Processing) ENGINE
=================================================================================

The NlpProcessor class implements several core NLP techniques:

A. TOKENIZATION
   Purpose: Break text into individual words/tokens
   Example: "How are you?" → ["how", "are", "you"]
   
   Implementation:
   - Convert to lowercase
   - Remove special characters
   - Split on whitespace
   - Filter empty tokens

B. STOP WORD REMOVAL
   Purpose: Remove common, less meaningful words
   Stop Words: a, an, the, is, are, and, or, but, for, from, with, etc.
   
   Benefit: Focuses analysis on important content words
   Example: "What is your pricing?" → ["your", "pricing"]

C. STEMMING
   Purpose: Reduce words to their root form
   Examples:
   - "running", "runs", "ran" → "run"
   - "integration", "integrated" → "integrat"
   - "services", "service" → "servic"
   
   Implementation: Suffix removal rules + lookup table
   Benefit: Matches variations of the same concept

D. SIMILARITY MATCHING (Jaccard Similarity)
   Purpose: Find FAQ question closest to user input
   Formula: similarity = |A ∩ B| / |A ∪ B|
            (intersection / union of token sets)
   
   Range: 0 to 1 (0 = no match, 1 = perfect match)
   Threshold: 0.4 (40% similarity required)
   Time Complexity: O(n) where n = number of FAQs
   
   Example:
   User: "What is the price?"
   FAQ:  "How much does it cost?"
   Similarity: 0.67 (67%) - Above threshold, returns answer

E. INTENT RECOGNITION
   Purpose: Understand user's goal from message
   
   Intent Types:
   - greeting      → "hello", "hi", "hey"
   - help_request  → "help", "how", "assist"
   - pricing       → "price", "cost", "pay"
   - contact       → "email", "phone", "reach"
   - onboarding    → "start", "signup", "begin"
   - gratitude     → "thanks", "appreciate"
   - general_query → other queries
   
   Implementation: Keyword matching with scoring

F. SENTIMENT ANALYSIS
   Purpose: Detect emotional tone of user message
   
   Sentiments: positive, negative, neutral
   
   Positive Words: good, great, excellent, amazing, love, best
   Negative Words: bad, terrible, hate, awful, poor, broken
   
   Implementation: Count occurrences and compare

G. ENTITY EXTRACTION
   Purpose: Identify important information in text
   
   Entities Extracted:
   - Email addresses: user@example.com
   - Phone numbers: 1-800-123-4567
   - Numbers: Any numeric values
   
   Implementation: Regular expression patterns

3. FAQ TRAINER & KNOWLEDGE BASE
=================================================================================

The FaqTrainer class manages the chatbot's knowledge:

FILE FORMAT: src/main/resources/faq_data.txt
Format: Q: question text|A: answer text (one per line)

Example:
Q: What services do you offer?|A: We offer AI chatbots and NLP solutions.
Q: How much does it cost?|A: Pricing starts at $99/month.

LOADING PROCESS:
1. Read file line by line
2. Parse Q&A pairs using "|" delimiter
3. Normalize questions (lowercase, trim, deduplicate)
4. Store in HashMap for O(1) lookup
5. Report statistics

MATCHING PROCESS:
1. User input → Normalize & process
2. Calculate similarity with each FAQ question
3. Return answer if similarity > 0.4 (40%)
4. Otherwise, use rule-based response

CUSTOMIZATION:
- Edit faq_data.txt to add/remove Q&A pairs
- Use /api/train endpoint to add new data via API
- Export to file using exportFaqToFile() method

4. REST API SPECIFICATION
=================================================================================

SERVER: http://localhost:8080
All responses: JSON format
CORS: Enabled for cross-origin requests

A. CHAT ENDPOINT
   URL: POST /api/chat
   Headers: Content-Type: application/json
   
   Request Body:
   {
     "message": "What services do you offer?"
   }
   
   Response:
   {
     "response": "We offer AI chatbots, natural language processing..."
   }
   
   Processing Flow:
   1. Validate message (not null/empty)
   2. Normalize input
   3. Process through Chatbot.processInput()
   4. Return JSON response

B. HEALTH CHECK ENDPOINT
   URL: GET /api/health
   
   Response:
   {
     "status": "healthy",
     "server": "AI Chatbot API",
     "version": "1.0",
     "timestamp": 1234567890,
     "faq_loaded": true
   }
   
   Purpose: Monitor server availability

C. DIAGNOSTICS ENDPOINT
   URL: GET /api/diagnostics
   
   Response:
   {
     "response_count": 5,
     "conversation_length": 5,
     "faq_database_size": 20,
     "faq_statistics": {
       "total_questions": 20,
       "avg_answer_length": 150,
       "avg_question_length": 45
     },
     "timestamp": 1234567890
   }
   
   Purpose: Get system statistics and performance data

D. TRAINING ENDPOINT
   URL: POST /api/train
   Headers: Content-Type: application/json
   
   Request Body:
   {
     "question": "What is your company?",
     "answer": "We are an AI company providing smart solutions."
   }
   
   Response:
   {
     "status": "trained",
     "question": "What is your company?"
   }
   
   Purpose: Add new Q&A pairs dynamically

5. RESPONSE GENERATION LOGIC
=================================================================================

User Input
    ↓
[Step 1: Process Input]
- Normalize text
- Tokenize and stem
- Analyze intent
- Detect sentiment
    ↓
[Step 2: Try FAQ Match]
- Calculate similarity with all FAQs
- If similarity ≥ 0.4 (40%) → Return FAQ answer
- Otherwise → Continue
    ↓
[Step 3: Rule-Based Response]
Based on detected intent:

INTENT: greeting
Response: "Hi! I'm here to assist you. What's your question?"

INTENT: help_request
Response: Context-dependent
- Contains "integrate" → API integration answer
- Contains "price" → Pricing information
- Contains "security" → Security details
- Otherwise → Generic help response

INTENT: pricing
Response: "Our pricing starts at $99/month for basic plans..."

INTENT: contact
Response: "You can reach us at support@example.com..."

INTENT: onboarding
Response: "Getting started is easy! Sign up for free account..."

INTENT: gratitude
Response: "You're welcome! Is there anything else I can help?"

INTENT: general_query
Response: Context-aware smart response based on keywords

[Step 4: Sentiment Adjustment]
If sentiment == negative:
  Append: "I apologize if you're experiencing issues..."
If sentiment == positive:
  Prepend: "Great! "

[Step 5: Return Response]
    ↓
Send to Frontend → Display to User

12. TROUBLESHOOTING GUIDE
=================================================================================

PROBLEM: Port 8080 Already in Use
SOLUTION: 
- Windows: netstat -ano | findstr :8080, then taskkill /PID [pid]
- Linux: lsof -i :8080, then kill -9 [pid]
- Or change PORT in ApiServer.java

PROBLEM: FAQ File Not Found
SOLUTION: 
- Ensure file: src/main/resources/faq_data.txt exists
- Check current working directory
- Use absolute path if relative fails

PROBLEM: Low Response Quality
SOLUTION:
- Expand FAQ database with more Q&A pairs
- Improve question phrasing
- Lower similarity threshold if too strict
- Add more keywords to rule-based responses

PROBLEM: Server Crashes
SOLUTION:
- Check Java version (11+ required)
- Check available memory
- Check file permissions
- Review logs for error messages

PROBLEM: Frontend Can't Connect
SOLUTION:
- Verify server is running
- Check port 8080 is open
- Verify CORS headers
- Check network connectivity
- Use proxy if behind firewall
